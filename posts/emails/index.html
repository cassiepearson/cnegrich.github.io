Sending secure emails (to me and others)

June 24, 2018

Why?

Sending secure emails is important if any sensitive information needs to be sent. This could include a vulnerability disclosure, anything with personally identifiable information (PII), or any other information that you consider sensitive.

How?

I will explain how to send a secure email to me specifically below. But many other people have written similar tutorials, here is a link to a Tech Republic article on it. We will be using gpg (Gnome Privacy Guard) for this tutorial. This is part of most linux operating systems by default and can be installed on windows as gpg4Win which offers some additional features.

To Me (Christopher Negrich)

You probably won’t have much of a reason to use this information to send emails to me, but this is fun and interesting and may help you in sending secure emails to others.

You have two options for sending me an encrypted email. The first is to use proton mail and email me at cnegrich@protonmail.com. This is the less secure option. The second option is to encrypt the email and then send it via proton mail. This is more secure.

Option one, make a proton mail account for free and send me an email. That was easy, but not as secure. This just allows you to send an end-to-end encrypted email to me. Meaning when it reaches my inbox, it will be in plaintext. This is fine for if you want to securely communicate less sensitive information, but if you want to send something with a bit more security, see option two.

Option two, still first make a proton mail account, you will encrypt the body of the email below and send it via proton mail. Please note that the subject line will be received unencrypted, keep that in mind when it is being sent. I provided my public key here, now all you need to do is encrypt a message using my public key. I will then use my private key that only I have to decrypt and read it. I will explain public key cryptography in much greater detail in a future post. The first thing we need to do to encrypt a message is import the key. Navigate here and save my key, then navigate to the directory you saved the key in and run the command gpg --import public_key.asc. Create a temporary file, ‘email’, and write your email. You can do this through the command nano email or vi email or vim email. You should secure delete the email file after creating the encrypted file by using a program such as bleachbit. A post going into more detail on secure deletion will be made in the future. Now, encrypt the file by running the command

gpg --output email.gpg --encrypt --recipient cnegrich@protonmail.com email

Now email me the string within the encrypted file, or the entire encrypted file. Don’t forget to go securely delete the original file. I will then similarly decrypt the email by running gpg --output email --decrypt email.gpg, and securely delete the file afterwards.

Both methods described above will help you to send more secure emails, but only the latter is sufficient when sending any highly sensitive information or PII. There are many other ways to send encrypted emails such as encrypting with ssh or sending an encrypted file with a pre-exchanged key or by sending a password locked (AES encrypted) 7zip file. This is just one way, I encourage you to explore others the next time you need to send someone sensitive information.

Generating Keys

This is a lot easier than you might think. Just run gpg --gen-key and enter the prompted information. After you have your key(s), you can view them my simply typing gpg -k and create a file of your public key by running gpg --armor --export EMAIL_ASSOCIATED_WITH_KEY > public_key.asc. You will then have a file called ‘public_key.asc’ containing your key in your current directory. That you can safely distribute to allow others to send secure emails to you. You may also want to include a link to this post if they need a short tutorial on how to do so.
