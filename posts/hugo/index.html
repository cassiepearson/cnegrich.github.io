Creating this site for yourself

June 24, 2018

Introduction

There are a lot of ways to create a site, but almost everyone agrees that writing plain HTML sucks. It is much better to generate the static HTML and add more advanced functionality through javascript using tools like React, Redux, and Babel. This site is just generated static HTML since there is no need for more advanced javascript functionality. To generate this site, I use Hugo. This is a pretty simple process. Download hugo, start a project, choose a theme, customize the config, and then write everything else in markdown. That sounds like a lot of steps, but it doesn’t take long. Once you have your site, you then need a domain to host it on, and some domain host to host it for you. I use namecheap for my domain name purchases, cloudflare to control my DNS (Domain Name Service) records after purchase (and enforce security), and finally github to host. This is a lot easier than it sounds and adds up to a total cost of around $12/year for this site. I will go into more detail in the next few sections.

Hugo

First, let’s make a site. For this, you will need your favorite text editor and the terminal, I recommend atom and oh-my-zsh respectively. Let’s start by installing hugo and choosing a theme. I am using the coder theme for this site. Once installed navigate to the directory where you want your new project and run hugo new site NAME this will create a site named NAME. If successful, hugo will print out a message with some instructions on next steps. Now lets insert the chosen theme, navigate into the new directory for your site and then into the themes directory. From here you can clone/branch the theme repository or download it and insert it. I would recommend renaming the theme you downloaded to be something simple if any additional version information was added to it. For example ‘coder-4.2310’ to just ‘coder’ since you will need to give this name in the hugo config.toml. Now that we have a theme, lets create an about page and a post directory. In order to do this, first move into the site directory by running cd name. Now hugo will recognize our theme and config file when creating new content. Now, before creating any content, we need to tell hugo that we want to use a theme, you can do this by running vim config.toml or nano config.toml if you are unfamiliar with vim. You can also do this using any text editor outside of the terminal. Add the lines baseurl = "/" and theme = "THEME_NAME", you may have more to add depending on how your theme is set up, check the theme set up instructions. Now, save and close the file and you are free to create new content. To create a new about page simple run hugo new about.md the ‘.md’ is for markdown, the type setting language that is compatible with hugo. To create a new directory of pages, such as a posts section, run hugo new posts/post1.md. To see your newly built, still empty, site you can run hugo server -D, this will build your site and run on localhost. Simply open a browser and navigate to http://localhost:1313 or 127.0.0.1 to view. The added ‘-D’ flag here tells hugo to include all pages marked as draft when building. This process will now run continually until stopped; this allows for you to make changes in the code and see the results immediately. From here, fill out your site with whatever you would like, and when ready, read the next section to learn more about hosting it.

Setting up the Domain and Hosting the Site

Purchasing a domain is always annoying. There are a lot of sites and you are never really certain who is the best. Personally, I like namecheap since they are cheap and more dependable than their name sounds. But I do not use them for anything except for registering the domain name, once I have the domain name, I move control over to cloudflare. You can create a free account and migrate this control relatively easily. But these decisions are yours, everyone has their favorites for registration and domain records. Finally, I use github pages to host my site for free under my custom domain name. I really like github and github pages is a very nice feature. Github is large and reliable so you can trust that your hosting service won’t have any problems. I will write the next section on how to set up your domain records and github to work with cloudflare and github pages. But as I said earlier, this decision is yours, choose a host and manager you trust.

This is the hardest part of setting up a site for anyone without networking experience. There are a lot of complicated sounding terms, but they are all simple, just stick with it. You only have to do this once if you do it correctly. First, lets go over a few of these common terms. DNS stands for domain name service, this is how a computer knows what IP address corresponds to a domain name. In order to access anything through the internet, you need its IP. This is almost always hidden from the end user by the DNS, since you simply type in the domain name and it redirects you to the corresponding IP stored in the DNS records. The DNS just looks up an IP given a domain name. Next is the host, this is the server that actually has the content of your site and a port open to the internet so others can access it. The IP of the host is the IP you need to put in your DNS records so everyone can easily find your site. Finally, a few in rapid succession, IP is the Internet Protocol, http is Hyper Text Transfer Protocol, and https is Secure Hyper Text Transfer Protocol. Respectively these govern how pages are accessed on the network that is the internet, control how pages are actually delivered to your computer, and control how pages are actually delivered to your computer while enforcing a minimum encryption standard. You don’t need to know any more detail than that, but if this interests you, I recommend starting by looking at the OSI (Open Systems Interconnection Reference) model and working your way from the the lowest level (layer 1) all the way to the highest (layer 7).

In order to host through github pages, we first need to create a new repository on github. You can do this via the command line if you want to, but since you will have to customize the settings on it, it is easier to just create this from github. Name this repository username.github.io, to let github know that it will be used for github pages. Now go under settings, customize to your preferences then navigate to the Github Pages section. Since the site it built using hugo, don’t worry about the theme choose, just go to custom domain and put in your purchased domain name, for me this is christophernegrich.com. If you purchased multiple domains for one website, just put the one you want as the main domain. For instance, chrisnegrich.com one of my other domains also forwards to christophernegrich.com, this will be set up in a few minutes. Also, leave off all https://www., we will deal with these in just a moment as well. Now, in order for our domain to refer to this github repository, we need to edit the DNS records. To do this, navigate to where you bought your domain, optionally (recommended) transfer control of the service to a free cloudflare account, and go to the DNS tab. For your domain to appear as the apex domain, the domain in the address bar when you are on the site, you need to configure new A records. Delete all current A records and add four new A records. For the name put your domain name, for example, christophernegrich.com and for the value put the the following IP addresses, one in each of the four records: 185.199.108.153 185.199.109.153 185.199.110.153 185.199.111.153 These IP addresses point to github’s servers. Now our domain will load our github repository when accessed. But, we are not done with the DNS records yet. We need to change or add a CNAME record for ‘www’, so our site is accessible from ‘www.domainname.com’. For this just create a new CNAME record with the name www and the value of domainname.com. If you have other domains that you want to point to this apex domain, you can set up similar CNAME records aliasing that domain to your primary domain. Alternatively, you could set up the same A name records and similar behavior will occur due to how github handles their pages. On cloudflare, you can configure other settings for your site including security rules. I would recommend looking through these other settings. Now, there is only one step left to have your site hosted on your custom domain through github pages. You need to push your built site to your github pages repository. Navigate to your site’s local directory in the terminal and run ‘hugo’ to build the site. Now navigate to the public directory and add all contents of this directory to a commit and commit to your new repository. Push up the commit and wait a few minutes for the page to update and your site will be up and hosted on your domain. Note that DNS records update much more slowly and after editing, it may take 12 hours for the changes to be in full effect. Now you have built your own site similar to this one for significantly cheaper than what squarespace or its competitors would charge. Feel free to contact me if you have questions about this process.
